Current Release

Version 0.0.1

17 June 2026 — Everything built in the Cognit project so far.

Cognit is a general-purpose programming language designed for AI and robotics, compiling to JavaScript. The entire project — compiler, runtime, CLI, playground, and tooling — was built from scratch.

1. Compiler

Three-phase pipeline: Lexer → Parser → Generator. Pure JavaScript, zero dependencies.

Lexer

Tokenizes Cognit source code into a stream of tokens with full position tracking.

  • Line / Column tracking — every token records its source position for precise error messages
  • Indent tracking — measures leading whitespace to emit Indent/Dedent tokens for block structure
  • Brace depth — when inside { }, indent tracking is disabled (braces manage nesting)
  • String interpolation detection — scans for {expr} inside double-quoted strings, splits into parts and sub-expressions
  • Operators — ** (exponentiation) and @ (matrix multiply) tokenized as Op
  • Keywords — let, var, fn, match, if, else, for, while, return, import, agent, pub, and, or, not, true, false, Some, None, Ok, Err, async, await, type
  • Comment skipping — # to end-of-line
  • Error reporting — throws LexError at line:col: message for invalid tokens, unterminated strings, indentation mismatches

Parser

Recursive descent parser with precedence climbing for expressions.

  • Program — sequence of statements separated by newlines
  • Bindings — let (compile to JS let, reassignable) and var (JS var)
  • Functions — fn name(params) -> expr or fn name(params) { body }; anonymous fn(params) -> expr
  • Pattern matching — match expr { pattern -> body ... } with literal, wildcard (_), binding, None/Some, Ok/Err, and tuple patterns
  • Expression-oriented if — if cond -> expr else if cond -> expr else -> expr compiles to ternary or IIFE
  • Pipe operator — a |> f(b) desugars to f(a, b); supports multi-line chains
  • Agent blocks — agent { key = value ... } parses as Agent AST node
  • Dict / Block disambiguation — { key: value } is a dict, { stmt; stmt } is a brace block
  • For / While — for x in iterable { body }, while cond { body }
  • Import — import "module" : name1, name2
  • Type annotations — optional : Type on params and return values
  • Semicolons — accepted as statement separators
  • Error reporting — ParseError at line:col: Expected X, got Y

Generator

Transpiles the AST to valid JavaScript, injects runtime functions via new Function(...keys, js).

  • Let bindings → let x = value;
  • Functions → function name(params) { return expr; }
  • Pipes → fn(piped, arg1, arg2) (piped value as first argument)
  • Match → IIFE with if/else if chain comparing against __match_val__
  • If/else → ternary for simple cases, IIFE for complex chains
  • String interpolation → concatenation with String() for embedded expressions
  • Agent → __cognit_agent__({ ... }) call
  • Logical operators — or → ||, and → &&

2. Language

Syntax Overview

// Bindings
let x = 42
var mutable = "hello"

// Functions
fn add(a, b) -> a + b
let double = fn(x) -> x * 2

// Pipes
let result = range(1, 10)
    |> filter(fn(x) -> x % 2 == 0)
    |> map(fn(x) -> x * x)
    |> sum

// Pattern matching
match value {
    0 -> "zero"
    1 -> "one"
    _ -> "many"
}

// Expression if
let grade = if score >= 90 -> "A"
            else if score >= 80 -> "B"
            else -> "F"

// String interpolation
print("Hello, {name}! You are {age}.")

// Lists and dicts
let list = [1, 2, 3]
let dict = {"key": "value", "count": 10}

// Loops
for i in range(0, 5) { print(i) }
while x > 0 { x = x - 1 }

// Operators
let exp = 2 ** 10       // exponentiation
let mm = a @ b          // matrix multiply
let eq = x == y         // comparison
let logical = a and b   // logical and/or/not

Comments

Single-line only: # this is a comment

Block Structure

Cognit supports two styles:

  • Indentation-based — like Python, uses Indent/Dedent tokens
  • Brace-based — { } blocks disable indent tracking

Agent Syntax

First-class syntactic construct for LLM agent configuration:

let assistant = agent {
    model = "gpt-4"
    temperature = 0.7
    instructions = "You are helpful"
}

Tensor Literals

Built-in functions create tensor objects:

let v = vec(1, 2, 3)           // 1D vector
let m = mat([[1,2],[3,4]])      // 2D matrix
let i = eye(3)                  // identity matrix
let z = zeros(2, 3)             // zero matrix
let o = ones(3)                 // ones vector

Error Formatting

Compile and parse errors display the exact source line with a caret pointer:

  examples/broken.cgn:5:3
    let x = 
      ^
  ParseError: Unexpected token '4'

REPL

Run cognit with no arguments. Type expressions line by line. .exit to quit.

5. Web Pages

Release Notes Page

Dark-themed homepage styled after TypeScript's release notes. Includes sidebar navigation, code blocks with syntax highlighting.

Playground

Full browser-based IDE for Cognit at play.html. Features:

  • Code editor — textarea with monospace font, dark theme
  • Synchronized gutter — line numbers scroll in perfect sync with the editor
  • Live compilation — auto-compiles on every keystroke, shows generated JavaScript
  • Run button — executes code and shows output in a dedicated tab
  • Error handling — compile errors in Errors tab, runtime errors in Output tab
  • Example selector — dropdown loads pre-written Cognit examples
  • Full compiler inline — the entire Lexer, Parser, and Generator are bundled as inline JavaScript

6. Examples

hello.cgn

Hello World, factorial, pipes, pattern matching, grade calculator:

print("Hello, World!")
print("2 + 2 =", 2 + 2)

fn factorial(n) {
    if n <= 1 -> 1
    else -> n * factorial(n - 1)
}
print("factorial(5) =", factorial(5))

let numbers = range(1, 21)
let doubled = numbers |> map(fn(x) -> x * 2)
let evens = numbers |> filter(fn(x) -> x % 2 == 0)

fn describe(n) {
    match n {
        0 -> "zero"
        1 -> "one"
        2 -> "two"
        _ -> "many"
    }
}

fn letter_grade(score) {
    match score {
        s if s >= 90 -> "A"
        s if s >= 80 -> "B"
        s if s >= 70 -> "C"
        s if s >= 60 -> "D"
        _ -> "F"
    }
}

advanced.cgn

Complex pipelines, pattern matching with guards, fibonacci, fizzbuzz, dict operations.

neural_net.cgn

Forward pass of a 2-layer neural network using tensor operations:

  • Sigmoid activation function
  • Matrix multiplication for weights
  • Batch processing (2 samples, 4 features → 3 hidden → 2 outputs)
  • Mean squared error loss calculation

robotics.cgn

Path planning and kinematics for robotics:

  • Waypoint distance calculation with pipes
  • Obstacle collision detection
  • Forward kinematics for a 2-link arm

sentiment_agent.cgn

AI agent pipeline for sentiment analysis:

  • Agent configuration blocks
  • Batch review processing
  • JSON export of results

8. Bugs Fixed

IssueDescription
Unary minusexp(-x) failed because parser checked isAt("-") but the token type is "Op". Fixed to check isAt("Op") && value === "-".
Multi-line let bindingsValues on the next line with indentation (let x =\n 42) failed because parser didn't skip Indent tokens. Added Indent skipping in parseLetBinding/parseVarBinding.
or / and operatorsGenerator emitted a or b instead of a || b. Added JS operator mapping in genBinaryOp.
Arrow + brace blockfn(x) -> { ... } treated the block as an expression, generating empty return;. Added { check in parseBlockOrExpr after arrow.
Gutter scroll syncLine numbers in playground gutter didn't stay aligned with editor text on scroll. Fixed by adding scroll event listener syncing gutter.scrollTop.
Trailing empty line in gutterGutter showed an extra line number for trailing \n. Fixed by excluding the last empty line from count.
